# DrawUI examples

Static browser demos for the dual `dist/` build. No bundler required — only a local HTTP server (ES modules do not load over `file://`).

Example pages link `examples/examples.css` after DrawUI CSS to re-enable document scrolling (DrawUI’s reset locks `html`/`body` to `overflow: hidden` for full-viewport apps).

## Run locally

Build first, then serve the `docs/` folder:

```powershell
npm run build
npx --yes serve docs -p 5173
```

Or use the combined script:

```powershell
npm run examples
```

Then open:

- **Docs hub:** http://localhost:5173/
- **Templates:**
  - 3D Viewer (workspace shell) — http://localhost:5173/examples/templates/
  - Site Logistics (AppShell + peers) — http://localhost:5173/examples/templates/site/
  - Ops Board (split layout + peers) — http://localhost:5173/examples/templates/ops/
- **Quick start:** http://localhost:5173/examples/min/

For GitHub Pages, set the site source to the `/docs` folder — [`index.html`](../index.html) is the entry.

> Previously at `/examples/min-components/` — that folder was renamed to `components/`.

## Min example

[`min/index.html`](./min/index.html) loads the bundled min build:

```html
<link rel="stylesheet" href="../../dist/drawui.min.css">
<script type="importmap">
{ "imports": { "drawui": "../../dist/drawui.min.js" } }
</script>
```

No peer packages. Theme CSS (`styles/themes/*.css`) is still linked separately.

After npm publish, swap the paths for CDN URLs:

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/drawui@0.2.0/dist/drawui.min.css">
<script type="module">
  import { DrawUI } from "https://cdn.jsdelivr.net/npm/drawui@0.2.0/dist/drawui.min.js";
</script>
```

## Live demo sources (internal)

[`components/demos.js`](./components/demos.js) and [`demos.full.js`](./components/demos.full.js) are the **source of truth** for live previews on API / TypeDoc pages. Edit those (and `catalog.manifest.json`), then run `npm run docs`.

The optional gallery shells (`components/index.html`, `full.html`) are developer tools for previewing demos locally — they are not linked from the docs hub.

| File | Role |
| --- | --- |
| `catalog.manifest.json` | Human-edited categories, peer category, and demo assignments |
| `catalog.json` | Generated by `npm run docs` |
| `demos.js` | Min preview renderers keyed by demo id |
| `demos.full.js` | Peer-dependent preview renderers (chart, spreadsheet, gantt, Showdown markdown) |
| `gallery.js` | Optional AppShell preview host (`{ build: "min" \| "full" }`) |
| `../spa-host.css` | Full-viewport host styles for SPA examples |

## Templates

Three demos with **different layouts** — not the same ribbon/workspace chrome:

| Demo | Path | Shape | Build |
| --- | --- | --- | --- |
| 3D Viewer | [`templates/`](./templates/) | `WorkspaceLayout` + ribbon + docked `WorkspacePanel`s | min |
| Site Logistics | [`templates/site/`](./templates/site/) | `AppShell` SPA (sidebar routes) | full — Chart, DataGrid, Gantt, Markdown |
| Ops Board | [`templates/ops/`](./templates/ops/) | Header + `SplitContainer` + tabs + `StatusBar` | full — Chart, DataGrid, NodeGraph |

### 3D Viewer (workspace shell)

Bonsai-web style: declarative `WorldComponent` → `#World` DOM, then `WorkspaceLayout` owns regions while each `WorkspacePanel` owns chrome. Undock → `FloatingWindow`.

| File | Role |
| --- | --- |
| [`index.html`](./templates/index.html) | Full-viewport entry |
| [`world-config.js`](./templates/world-config.js) | Region ids + ribbon `moduleId`s |
| [`app.js`](./templates/app.js) | Mount world, panels, viewport card |
| [`templates.css`](./templates/templates.css) | Host chrome |

### Site Logistics / Ops Board

Both load `drawui.full.js` with the same peer import map as [`components/full.html`](./components/full.html) (Chart.js, ag-grid, jsGantt, Showdown).

- **Site** — construction plant bookings, timesheets, delivery windows, look-ahead Gantt, daily briefing markdown.
- **Ops** — alert rail + metrics/incidents/topology tabs (no workspace ribbon).

Open any template URL above after `npm run examples` (run `npm run build` first so `docs/dist/drawui.full.js` exists).
